## Deskripsi
Pada proyek tugas besar, salah satu design pattern yang digunakan adalah Singleton Pattern, yang terdapat pada file frontend/src/lib/auth-client.ts.

import 'dotenv/config';
import { createAuthClient } from 'better-auth/react';

export const authClient = createAuthClient({
  baseURL: process.env.NEXT_PUBLIC_BETTER_AUTH_URL,
  basePath: '/api/v1/auth',
});

**Penjelasan Design Pattern**

Kode di atas menggunakan konsep Singleton, yaitu memastikan hanya ada satu instance dari authClient yang digunakan di seluruh aplikasi.

Pada environment frontend seperti Next.js, module yang di-import akan di-cache oleh sistem bundler. Artinya, ketika authClient di-import di beberapa file berbeda, yang digunakan tetap instance yang sama, bukan membuat objek baru setiap kali import.

Hal ini penting karena:

Menghindari pembuatan instance berulang
Menjaga konsistensi session/authentication
Mengurangi penggunaan memori yang tidak perlu
Menghindari konflik data login atau cookie

Jika tidak menggunakan pola ini dan setiap komponen membuat client sendiri, maka bisa terjadi duplikasi instance yang berpotensi menyebabkan masalah pada autentikasi aplikasi.

**Kesimpulan**

Dengan menggunakan Singleton Pattern, authClient hanya dibuat sekali dan digunakan kembali di seluruh aplikasi, sehingga lebih efisien, konsisten, dan aman untuk sistem autentikasi.
