## Soal
Diberikan dua kode yang sama-sama melakukan operasi pembagian. Pertama menggunakan asersi, kedua menggunakan eksepsi.
```
const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}
```
```
function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}
```
Menurutmu, kapankah kita saatnya menggunakan asersi atau eksepsi untuk fungsi seperti ini di atas? Apakah kita harus sepenuhnya asersi, atau sepenuhnya eksepsi? Lakukan riset dan berikan jawabannya dalam bentuk esai minimal 300 kata.
## Jawaban
Dalam pemrograman, asersi (assertion) dan eksepsi (exception) sama-sama digunakan untuk menangani kondisi error, tetapi keduanya memiliki tujuan yang berbeda. Pada contoh fungsi `divide(a, b)`, penggunaan asersi maupun eksepsi sebenarnya memiliki peran masing-masing dan tidak sebaiknya digunakan secara sepenuhnya salah satu saja.

Asersi biasanya digunakan untuk memeriksa kondisi yang “seharusnya tidak pernah salah” apabila program ditulis dengan benar. Asersi lebih cocok dipakai saat proses development atau debugging untuk mendeteksi kesalahan logika programmer. Ketika kondisi asersi gagal, program akan langsung dihentikan agar developer segera mengetahui adanya bug pada kode. Microsoft menjelaskan bahwa assert digunakan untuk kondisi yang harus selalu benar dan bukan untuk error yang memang mungkin terjadi saat runtime. ([Microsoft Learn][1])

Sebaliknya, eksepsi digunakan untuk menangani error yang memang mungkin terjadi saat program dijalankan, seperti input pengguna yang salah, file tidak ditemukan, atau pembagian dengan nol. Exception dirancang agar program dapat menangani kesalahan secara lebih aman tanpa langsung menghentikan aplikasi. Dengan adanya `try...catch`, program masih memiliki kesempatan melakukan recovery atau menampilkan pesan error yang jelas kepada pengguna. ([Microsoft Learn][2])

Pada kasus fungsi `divide(a, b)`, pengecekan tipe data dan pembagian dengan nol lebih tepat menggunakan exception dibanding assert. Hal ini karena kesalahan tersebut bisa saja berasal dari input pengguna atau sistem lain, sehingga termasuk runtime error yang wajar terjadi. Jika hanya menggunakan assert, maka ketika assert dinonaktifkan pada mode production, validasi tersebut bisa hilang dan menyebabkan program menjadi tidak aman. ([CodingTechRoom][3])

Namun, bukan berarti assert tidak berguna. Assert tetap baik digunakan untuk memastikan invariant atau asumsi internal program, misalnya memastikan variabel tertentu tidak bernilai null pada bagian kode yang secara logika memang tidak mungkin null. Banyak praktisi juga menyebut assert sebagai alat “fail-fast” untuk mendeteksi bug sejak awal pengembangan. ([Stack Overflow][4])

Menurut saya, pendekatan terbaik adalah menggabungkan keduanya sesuai kebutuhan. Assertion digunakan untuk debugging dan validasi internal developer, sedangkan exception digunakan untuk menangani error runtime yang dapat terjadi dalam penggunaan nyata aplikasi. Dengan kombinasi ini, kode menjadi lebih aman, mudah dipelihara, dan lebih profesional dalam menangani kesalahan.
