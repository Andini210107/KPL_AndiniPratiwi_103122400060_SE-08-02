## Soal
Dari dua kode di bawah ini, mana yang kamu ingin cari masalahnya dan perbaiki di tengah-tengah malam, katakanlah jam 1 malam? Mengapa?
```javascript
function processUser(user) {
  if (user) {
    if (user.isActive) {
      if (user.hasPermission) {
        return doSomething(user)
      }
    }
  }
  return null
}
```
```javascript
function processUser(user) {
  if (!isValidCandidate(user)) return null;
  return doSomething(user);
}

function isValidCandidate(user) {
  return user && user.isActive && user.hasPermission;
}
```

## Jawaban
Jika harus memilih salah satu kode untuk dicari masalahnya dan diperbaiki pada jam 1 malam, saya akan memilih kode yang kedua.

Alasannya karena kode kedua lebih sederhana dan mudah dibaca. Logika pengecekan dipisahkan ke dalam fungsi isValidCandidate(), sehingga fungsi processUser() hanya fokus pada proses utama. Dengan struktur seperti ini, saya bisa lebih cepat memahami alur program dan menemukan sumber masalah jika terjadi bug.

Sebaliknya, kode pertama menggunakan banyak nested if yang membuat alur logika lebih panjang dan sulit dibaca. Saat kondisi yang diperiksa semakin banyak, proses debugging juga akan menjadi lebih rumit karena harus menelusuri setiap percabangan satu per satu.

Menurut saya, kode yang lebih ringkas dan modular akan lebih mudah dipelihara, diuji, dan diperbaiki, terutama ketika sedang lelah atau bekerja pada malam hari. Oleh karena itu, saya lebih memilih kode kedua karena lebih efisien dari sisi keterbacaan dan debugging.
